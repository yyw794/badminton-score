# LLM 推理排阵系统

## 概述

这是基于大模型推理能力的羽毛球排阵系统，作为传统算法的补充方案。通过模拟人类专家的思考过程，实现更智能、更灵活的排阵策略。

## LLM 推理的核心思路

### 1. 分层优先级策略

LLM 排阵采用人类专家的思考方式，按优先级处理约束：

```
优先级 1（最高）：固定场次要求（未完成时大幅减分）
优先级 2：未达到目标场次（5 场）的球员
优先级 3：连续轮空 1 轮的球员（防止冷场）
优先级 4：外援球员（小幅加分，降低优先级）
优先级 5：达到最大场次的球员（大幅加分，靠后）
优先级 6：固定搭档组合（减分，优先安排）
```

### 2. 轮次交替策略

通过交替不同类型的优先级，确保比赛类型多样化：

- 第 1、4、7 轮：女双优先
- 第 2、5、8 轮：混双优先
- 第 3、6、9 轮：男双优先

### 3. 上下文感知

每轮排阵时，LLM 会考虑：

- 球员的历史上场记录
- 球员是否连续轮空
- 当前轮次已安排的球员（确保不重复）
- 球员的剩余可用场次

### 4. 动态调整

当首选类型无法安排时，自动降级到备选方案：

```
首选类型 → 次选类型 → 男双替补 → 临时组合
```

## 核心规则保证

### 规则 1：同一轮次不重复

每个球员在所有场地（1 号、2 号、3 号）只能出现一次。

**实现方式**：每轮安排前检查 `current_round_matches`，确保球员未在本轮已上场比赛。

### 规则 2：防止冷场

球员不能连续轮空 2 轮，第 3 轮必须安排上场。

**实现方式**：
- 跟踪 `player_bye_history`（轮空历史）
- 计算 `consecutive_byes`（连续轮空数）
- 当连续轮空 ≥ 1 时，标记为 `must_play`，优先安排

### 规则 3：外援优先级降低

外援球员最大场次 5 场，内部员工 6 场。

**实现方式**：评分算法中，外援球员 +50 分（降低优先级）。

## 评分算法详解

```python
def calculate_match_priority(match):
    score = 0
    
    for player in match_players:
        # 1. 固定场次要求（未完成 -1000 分/场）
        if fixed_games_remaining > 0:
            score -= fixed_games_remaining * 1000
        
        # 2. 未达目标场次（-200 分/场）
        if games < 5:
            score -= (5 - games) * 200
        
        # 3. 连续轮空（-500 分）
        if consecutive_byes >= 1:
            score -= 500
        
        # 4. 外援球员（+50 分）
        if is_guest_player:
            score += 50
        
        # 5. 已达最大场次（+2000 分）
        if games >= max_games:
            score += 2000
    
    # 6. 固定搭档（-权重*30 分）
    if is_fixed_partner:
        score -= partner_weight * 30
    
    return score
```

**评分越低，越优先被选中**。

## 与传统算法的区别

| 维度 | 传统算法 | LLM 推理 |
|------|----------|----------|
| 核心思路 | 贪心算法 + 启发式规则 | 模拟人类专家思考 |
| 优先级管理 | 硬编码 if-else | 评分系统动态调整 |
| 上下文感知 | 有限状态跟踪 | 完整历史记录 |
| 灵活性 | 固定规则 | 可动态调整权重 |
| 可解释性 | 需要查看代码 | 评分即解释 |
| 扩展性 | 需修改代码 | 调整评分权重即可 |

## LLM 推理的优势

1. **更人性化**：模拟人类专家的决策过程
2. **更灵活**：通过调整评分权重即可改变策略
3. **更透明**：每个选择的评分都可以追溯
4. **更易维护**：核心逻辑与评分逻辑分离
5. **可扩展**：新增约束只需添加评分项

## 常见问题

### Q: LLM 排阵和传统算法哪个更好？

A: 两种方法各有优劣：
- 传统算法：稳定、可预测、执行快
- LLM 推理：灵活、人性化、易调整

建议同时运行两种算法，选择更满意的方案。

### Q: 如何调整排阵策略？

A: 修改 `_calculate_match_priority()` 方法中的评分权重：
- 增加固定场次权重：`score -= remaining * 2000`（更优先）
- 降低外援优先级：`score += 100`（更靠后）

### Q: 可以只使用 LLM 排阵吗？

A: 可以。`llm_scheduler.py` 是独立的，不依赖传统算法。

## 技术细节

### 数据结构

```python
# 比赛池
mixed_pool: List[Tuple[Tuple[str, str], Tuple[str, str]]]
# 示例：[(('林锋', '田茜'), ('王小波', '李祺祺')), ...]

# 已选比赛
scheduled_matches: List[Dict]
# 示例：[{

#     "match": (('林锋', '田茜'), ('王小波', '李祺祺')),
#     "round": 1,
#     "court": 1

# 球员状态
player_games: Dict[str, int]  # 已赛场次
player_bye_history: Dict[str, List[bool]]  # 轮空历史
```

### 复杂度分析

- 时间复杂度：O(R × C × M)
  - R = 轮次数
  - C = 场地数
  - M = 比赛池大小
- 空间复杂度：O(P × R)
  - P = 球员数
  - R = 轮次数
