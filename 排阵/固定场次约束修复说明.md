# 固定场次约束修复说明

## 问题描述

之前发现提前离场的球员（严勇文、崔倩男）虽然设置了 `fixed_games: 5`，但两种算法都给他们排了 6 场比赛，违反了固定场次约束。

## 问题根源

### 原因 1：防止冷场规则的例外处理不当

原代码逻辑：
```python
# 已达到固定场次，可以不参加，但如果必须上场（连续轮空），则仍允许参加
if fixed_games is not None and player_games[player] >= fixed_games:
    if must_play:  # 连续轮空 1 轮
        pass  # 仍然允许上场
    else:
        return False
```

这导致即使球员达到了固定场次，如果连续轮空 1 轮，仍然会被强制安排上场。

### 原因 2：总比赛数未根据可用人次动态调整

原代码固定使用 `总比赛数 = 8 场 × 场地数 = 24 场`，但当有球员只能打 5 场时，实际可用人次不足以支撑 24 场比赛。

**计算**：
- 严勇文：5 场
- 崔倩男：5 场
- 其他 8 个内部球员：8 × 7 = 56 场
- 2 个外援：2 × 6 = 12 场
- **总计**：78 人次 = 19.5 场比赛

所以实际最多只能安排 19-21 场比赛，而不是 24 场。

## 修复方案

### 修复 1：严格执行 fixed_games 约束

**传统算法** (`lineup_scheduler.py`):
```python
# 【固定场次规则】严格执行 fixed_games 限制
# 有 fixed_games 约束的球员，达到固定场次后禁止参赛（不享受冷场规则例外）
if fixed_games is not None and player_games[player] >= fixed_games:
    return False
```

**LLM 算法** (`llm_scheduler.py`):
```python
# 【Fixed Games Rule】Strictly enforce fixed_games limit
# Players with fixed_games CANNOT exceed their fixed games (no exception for byes)
if fixed_games is not None and self.player_games[player] >= fixed_games:
    return False
```

### 修复 2：有 fixed_games 的球员不适用防止冷场规则

**传统算法**:
```python
def must_player_play(player):
    """
    检查球员是否必须上场（因为已经连续轮空 1 轮）。
    注意：有 fixed_games 约束的球员不适用此规则，
    因为他们有固定的场次安排，会提前离场。
    """
    constraint = get_constraint(player)
    fixed_games = constraint.get("fixed_games")
    
    # 有 fixed_games 约束的球员不适用防止冷场规则
    if fixed_games is not None:
        return False
    # ...
```

**LLM 算法**:
```python
def _must_player_play(self, player: str) -> bool:
    """
    Check if a player must play to avoid consecutive byes.
    
    Note: Players with fixed_games constraint are EXCLUDED from this rule,
    because they have a fixed number of games to play and will leave early.
    """
    constraint = self._get_constraint(player)
    fixed_games = constraint.get("fixed_games")
    
    # Players with fixed_games are excluded from anti-cold rule
    if fixed_games is not None:
        return False
    # ...
```

### 修复 3：评分算法中添加巨额惩罚

**传统算法**:
```python
# 优先级 1：已达到固定场次的球员 = 禁止参赛（巨额加分）
if fixed_games is not None and games >= fixed_games:
    score += 10000  # 巨额加分，确保不会被选中
```

**LLM 算法**:
```python
# Priority 1: FORBIDDEN if already reached fixed_games (huge penalty)
if fixed_games is not None and games >= fixed_games:
    score += 10000  # Huge penalty - should not be selected
```

### 修复 4：动态计算总比赛数

**两种算法都添加了以下逻辑**:
```python
# Calculate actual available player-games
total_available_games = 0
for p in all_players:
    constraint = PLAYER_CONSTRAINTS.get(p, {})
    fixed_games = constraint.get("fixed_games")
    if fixed_games is not None:
        total_available_games += fixed_games
    else:
        total_available_games += get_max_games_for_player(p)

# Each match requires 4 player-games
max_possible_matches = total_available_games // 4
target_matches = MATCHES_PER_COURT * court_count
total_matches = min(max_possible_matches, target_matches)
```

## 使用建议

### 如果有更多球员需要固定场次约束

在 `PLAYER_CONSTRAINTS` 中添加：
```python
PLAYER_CONSTRAINTS = {
    "严勇文": {"fixed_games": 5, "early_departure": True},
    "崔倩男": {"fixed_games": 5, "early_departure": True},
    "张三": {"fixed_games": 3, "early_departure": True},  # 新增
}
```

### 如果希望某球员不受固定场次限制

不要设置 `fixed_games`，只设置 `early_departure`：
```python
PLAYER_CONSTRAINTS = {
    "李四": {"early_departure": True},  # 会提前离场，但没有固定场次要求
}
```

### 如果希望严格执行防止冷场规则

确保球员没有 `fixed_games` 约束，这样他们会正常参与防止冷场规则：
```python
PLAYER_CONSTRAINTS = {
    "王五": {},  # 没有 fixed_games，会正常参与防止冷场规则
}
```

## 技术细节

### 优先级顺序（评分系统）

1. **已达到固定场次** = 禁止参赛（+10000 分，巨额惩罚）
2. **固定场次未完成** = 优先安排（-1000 分/场）
3. **未达目标场次** = 优先安排（-200 分/场）
4. **连续轮空 1 轮** = 中等优先（-500 分，不适用于 fixed_games 球员）
5. **外援球员** = 降低优先级（+50 分）
6. **达到最大场次** = 靠后安排（+2000 分）
7. **固定搭档** = 优先安排（-权重×30 分）

### 核心规则保证

1. ✅ **同一轮次不重复**：每个球员在所有场地只能出现一次
2. ✅ **固定场次严格执行**：有 fixed_games 的球员达到场次后禁止参赛
3. ✅ **防止冷场规则**：无 fixed_games 的球员不能连续轮空 2 轮
4. ✅ **外援优先级降低**：外援最大 5-6 场，内部员工最大 7 场
