# 羽毛球排阵系统 - LLM 推理方案

## 概述

本项目现在支持两种排阵方式：

1. **传统算法** (`排阵/lineup_scheduler.py`) - 基于贪心算法和启发式规则
2. **LLM 推理** (`排阵/llm_scheduler.py`) - 基于大模型推理能力的智能排阵

## 快速开始

### 方式一：只运行 LLM 排阵

```bash
cd 排阵
python llm_scheduler.py
```

输出：`排阵/对阵表_LLM.xlsx`

### 方式二：对比两种算法

```bash
cd 排阵
python run_comparison.py
```

输出：
- `排阵/对阵表_传统.xlsx` - 传统算法结果
- `排阵/对阵表_LLM.xlsx` - LLM 推理结果

### 方式三：只运行传统算法（原有方式）

```bash
cd 排阵
python lineup_scheduler.py
```

输出：`排阵/对阵表.xlsx`

## LLM 推理的核心优势

| 维度 | 传统算法 | LLM 推理 |
|------|----------|----------|
| 核心思路 | 贪心算法 + 硬编码规则 | 模拟人类专家思考 |
| 优先级管理 | if-else 硬编码 | 评分系统动态调整 |
| 上下文感知 | 有限状态跟踪 | 完整历史记录 |
| 灵活性 | 需修改代码 | 调整评分权重即可 |
| 可解释性 | 需要查看代码 | 评分即解释 |
| 扩展性 | 较复杂 | 简单（添加评分项） |

## LLM 推理的评分策略

```
优先级 1（最高）：固定场次要求（未完成 -1000 分/场）
优先级 2：未达到目标场次（5 场）(-200 分/场)
优先级 3：连续轮空 1 轮（防止冷场）(-500 分)
优先级 4：外援球员（降低优先级）(+50 分)
优先级 5：达到最大场次 (+2000 分)
优先级 6：固定搭档组合（优先安排）(-权重*30 分)
```

**评分越低，越优先被选中**。

## 核心规则保证

两种算法都严格保证以下规则：

### 规则 1：同一轮次不重复
每个球员在所有场地（1 号、2 号、3 号）只能出现一次。

### 规则 2：防止冷场
球员不能连续轮空 2 轮，第 3 轮必须安排上场。

### 规则 3：外援优先级降低
外援球员最大场次 5 场，内部员工 6 场。

## 如何调整 LLM 排阵策略

修改 `llm_scheduler.py` 中的 `_calculate_match_priority()` 方法：

```python
# 更重视固定场次要求
score -= remaining * 2000  # 原来是 1000

# 更降低外援优先级
score += 100  # 原来是 50

# 更严格防止冷场
score -= 1000  # 原来是 500
```

## 技术细节

### 数据结构

```python
# 比赛池
mixed_pool: List[Tuple[Tuple[str, str], Tuple[str, str]]]
# 示例：[(('林锋', '田茜'), ('王小波', '李祺祺')), ...]

# 已选比赛
scheduled_matches: List[Dict]
# 示例：[{
#     "type": "混双",
#     "match": (('林锋', '田茜'), ('王小波', '李祺祺')),
#     "round": 1,
#     "court": 1
# }, ...]

# 球员状态
player_games: Dict[str, int]  # 已赛场次
player_bye_history: Dict[str, List[bool]]  # 轮空历史
```

### 复杂度分析

- 时间复杂度：O(R × C × M)
  - R = 轮次数
  - C = 场地数
  - M = 比赛池大小
- 空间复杂度：O(P × R)
  - P = 球员数
  - R = 轮次数

## 常见问题

### Q: LLM 排阵和传统算法哪个更好？

A: 两种方法各有优劣：
- 传统算法：稳定、可预测、执行快
- LLM 推理：灵活、人性化、易调整

建议同时运行两种算法，选择更满意的方案。

### Q: 可以只使用 LLM 排阵吗？

A: 可以。`llm_scheduler.py` 是独立的，不依赖传统算法。

### Q: 如何查看详细推理过程？

A: 运行 `llm_scheduler.py` 时会输出每轮的排阵进度和最终统计。
