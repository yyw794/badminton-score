# 羽毛球比分图片解析规则

> 用于每周图片解析、统计分析和排名

## 比赛基本信息

- **赛制**: 15分制，达到15分的一方获胜
- **比赛类型**: 单打、男双、女双、混双（统计个人数据）
- **场地数量**: 根据图片实际场地数量记录（通常2个或3个场地）
- **比赛记录格式**: 每轮比赛包含多个场地，每个场地1场比赛
- **局数**: 每场比赛分2局，分别记录在比分A和比分B
- **单打判断**: 如果"对阵A"或"对阵B"里只有1个名字，则为单打

## 表格字段说明

| 字段 | 说明 |
|------|------|
| 轮次 | 第1轮、第2轮... |
| 场地 | 1号场地、2号场地（仅记录实际存在的场地，不要臆造不存在的场地） |
| 对阵A | 主队对阵信息，格式：单打为`选手1`，双打为`选手1 / 选手2` |
| 比分A | 第一局比分，格式：`A: B`，表示A队第一局得分:B队第一局得分 |
| 比分B | 第二局比分，格式：`A: B`，表示A队第二局得分:B队第二局得分 |
| 对阵B | 客队对阵信息，格式：单打为`选手1`，双打为`选手1 / 选手2` |

## 比分解析规则

### 比分格式

- 比分A表示：对阵A选手第一局得分 : 对阵B选手第一局得分
- 比分B表示：对阵A选手第二局得分 : 对阵B选手第二局得分
- 例如：比分A `15:13` 表示对阵A第一局15分，对阵B第一局13分

### 获胜判断

- 达到15分且比分高于对方的一方获胜该局
- 如比分A为 `15:13`，则对阵A赢得第一局
- 如比分A为 `13:15`，则对阵B赢得第一局

### 手写数字易混淆识别

| 容易混淆 | 区分方法 |
|----------|----------|
| 1 和 7 | 看起笔方向，7通常有横杠 |
| 3 和 8 | 3开口朝上，8上下两个圈 |
| 5 和 6 | 5有横杠，6是圆圈带尾巴 |
| 4 和 9 | 4是三角形+直线，9是圆圈带尾巴 |
| 0 和 6 | 0是椭圆，6有向下弯的尾巴 |
| 11 和 17 | 11中间有分隔，17中间分开 |
| 13 和 17 | 13的7短，17的7长且有横杠 |
| 10 和 17 | 10的1和0分开，17连在一起 |

### 比分异常处理

- **比分中出现超过15分的数字**（如16、18等）：视为书写错误，结合上下文修正
- **比分相同的情况**（如 `15:15`）：不可能，需要用户确认
- **手写字迹不清晰**：标注为空字符串 `""`，并提示用户确认
- **不确定的比分**：必须让用户确认后再记录
- **每局必须有一方达到15分**：如果双方都未达到15分，该比分无效需确认

### 解析后自查清单

解析完成后，必须逐项核对：

1. **比分自查**：每一局是否有一方达到15分？
2. **对阵自查**：每个选手名字是否在选手名单中？
3. **轮次自查**：每个轮次是否包含所有场地？
4. **局数自查**：每场比赛是否都有2局比分？
5. **逻辑自查**：每场比赛是否至少有1人出场（不重复）？
6. **特殊人名**：是否有人名写错（参考特殊人名纠正表）？

## 输出格式

### match_data.json

```json
{
  "match_date": "YYYY-MM-DD",
  "description": "比赛描述",
  "format": "15分/局，2局",
  "matches": [
    {
      "round": 轮次数字,
      "court": 场地数字,
      "type": "单打/男双/女双/混双",
      "team_a": ["选手1", "选手2"],
      "team_b": ["选手1"],
      "score_a": "第一局比分 A:B",
      "score_b": "第二局比分 A:B",
      "notes": "异常情况说明（可选）"
    }
  ]
}
```

### 重要规则

1. **场地数量必须与图片一致**，不要臆造不存在的场地
2. **每场比赛必须包含score_a和score_b两个比分**
3. **比分必须有一方达到15分**，且比分高于对方
4. **不确定/看不清的比分**：设为空字符串 `""`，并在notes中说明
5. **对阵信息不完整或重复**：标注为空字符串 `""`，并在notes中说明
6. **所有选手名字必须正确**，出现错误写法立即纠正

## 统计分析

### 个人统计

- 计算每个人的胜负场次和胜率
- 根据胜率进行总排名

### 分类统计

- 分别统计单打、男双、混双、女双场景下的胜率
- 按胜率排序展示

## 工具使用（从根目录执行）

每周图片解析必须使用以下脚本进行 OCR 识别，生成 Markdown 结果后人工核对并转换为 `match_data.json`。

**所有脚本从项目根目录执行**（即 `pingan_tech_badminton_team` 目录）：

### 1. OCR 解析

```bash
python3 scores/paddleocr_vl.py scores/20260622/20260622.jpg
```

输出目录会自动生成为 `scores/20260622/output_20260622`，包含：
- `paddleocr_raw.jsonl`：OCR 原始结果（用于回溯）
- `doc_0.md`：结构化 Markdown 表格

### 2. 自动验证

```bash
python3 scores/verify_ocr.py scores/20260622/output_20260622/doc_0.md
```

验证规则：
- 每局比分必须有一方达到 15 分
- 获胜方必须达到 15 分（检测手写识别错误，如 `15` 被识别成 `6`）
- 选手名字必须在已知名单中（发现不在名单中的名字会告警）
- 人名自动纠正（如"李棋棋"→"李祺祺"）

输出示例：
```
✅ 全部 16 场比赛验证通过，无需确认
```
或
```
⚠️ 发现 2 处需要确认:
  - R1C1 第1局：6:15 获胜方 6 分未达到 15 分，可能是手写识别错误
  - R8C1 对阵A：'陈小兵' 不在已知名单中
```

### 3. 统计分析

```bash
python3 scores/calculate_stats.py
```

自动查找最近的 `match_data.json` 文件并生成排名。

### 工作流程

1. 使用 `paddleocr_vl.py` 解析图片，生成 Markdown 结果
2. 运行 `verify_ocr.py` 自动验证 OCR 解析结果
3. 如有警告项，对照原图确认；如无警告，直接信任 OCR 结果
4. 根据验证结果将 Markdown 转换为 `match_data.json`
5. 运行 `calculate_stats.py` 生成排名统计
