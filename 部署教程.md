
## 🚀 部署步骤

### 第 1 步：部署后端 API 到阿里云函数计算

#### 1.1 开通阿里云函数计算

1. 访问 [阿里云函数计算](https://fc.console.aliyun.com/)
2. 登录/注册阿里云账号（需要实名认证）
3. 开通函数计算服务（新用户有免费额度）

#### 1.2 创建函数

1. 进入 **函数列表** → **创建函数**
2. 选择 **从零开始创建**
3. 配置如下：
   - **函数名称**: `badminton-score-api`
   - **运行环境**: `Python 3.10`
   - **代码上传方式**: `本地上传`
   - **Handler**: `index.handler`

#### 1.3 上传代码

1. 将 `api/` 目录下的文件打包成 ZIP：
   ```bash
   cd api
   zip -r function.zip index.py requirements.txt s.yaml
   ```

2. 在函数配置页面上传 `function.zip`

#### 1.4 配置环境变量（重要！）

在函数配置页面，添加以下环境变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `TIDB_HOST` | `gateway01.ap-southeast-1.prod.aws.tidbcloud.com` | TiDB 主机 |
| `TIDB_PORT` | `4000` | TiDB 端口 |
| `TIDB_USER` | `8AXKPCJGEHG5Qqv.root` | TiDB 用户名 |
| `TIDB_PASSWORD` | `7iv93aerRyGckxqw` | TiDB 密码 |
| `TIDB_DATABASE` | `badminton` | 数据库名 |

⚠️ **注意**：数据库密码会保存在服务端，用户无法看到

#### 1.5 配置触发器

1. 在函数页面，点击 **触发器** → **创建触发器**
2. 选择 **HTTP 触发器**
3. 配置：
   - **触发器名称**: `http-trigger`
   - **请求方法**: `GET, POST, PUT, DELETE, OPTIONS`
   - **认证方式**: `无需认证`（或根据需要配置）

4. 创建后，复制 **触发器 URL**（类似 `https://xxx.fc.aliyuncs.com/...`）

### 第 2 步：初始化数据库

#### 2.1 在 TiDB 创建数据库

使用 MySQL 客户端或 TiDB Cloud 控制台执行：

```sql
CREATE DATABASE IF NOT EXISTS badminton;
```

#### 2.2 调用初始化接口

在浏览器访问：
```
https://your-api-url.fc.aliyuncs.com/init
```

看到以下响应表示成功：
```json
{"success": true, "message": "数据库表初始化完成"}
```

### 第 3 步：创建活动数据

#### 方式 1：使用 API 工具（推荐）

使用 Postman 或 curl 创建：

```bash
curl -X POST https://your-api-url.fc.aliyuncs.com/events \
  -H "Content-Type: application/json" \
  -d '{
    "name": "2026 马年首秀战",
    "court_count": 3,
    "matches": [
      {"id": "m1", "round": 1, "court": 1, "type": "混双", "teamA": ["林锋", "李祺祺"], "teamB": ["王小波", "谢卓珊"]},
      {"id": "m2", "round": 1, "court": 1, "type": "男双", "teamA": ["罗蒙", "陈顺星"], "teamB": ["陈小洪", "卢志辉"]},
      ...（参考 data.json 中的所有比赛）
    ]
  }'
```

响应：
```json
{"success": true, "data": {"id": "生成的活动 ID"}}
```

#### 方式 2：使用现有 data.json

运行以下 Python 脚本导入：

```bash
cd api
python import_from_json.py ../docs/data.json
```

---

### 第 4 步：修改前端配置

编辑 `docs/app.js` 文件：

```javascript
// 第 7-9 行
const API_BASE_URL = 'https://your-api-url.fc.aliyuncs.com';  // 替换为你的 API URL
const EVENT_ID = '你的活动 ID';  // 替换为创建的活动 ID

// 第 12 行
const ENABLE_CLOUD_SYNC = true;  // 改为 true 启用云端同步
```

---

### 第 5 步：部署前端到 GitHub Pages

1. 提交修改到 Git：
   ```bash
   git add docs/app.js
   git commit -m "启用云端同步"
   git push
   ```

2. 在 GitHub 仓库设置中，确保 GitHub Pages 已启用：
   - Settings → Pages
   - Source: `Deploy from a branch`
   - Branch: `main` / `docs`

## 📊 API 接口说明

| 接口 | 方法 | 说明 |
|------|------|------|
| `/init` | GET | 初始化数据库表 |
| `/events` | GET | 获取活动列表 |
| `/events` | POST | 创建活动 |
| `/events/{id}` | GET | 获取活动详情（含比赛） |
| `/events/{id}` | PUT | 更新活动 |
| `/events/{id}/matches` | GET | 获取比赛列表 |
| `/matches/{id}` | PUT | 更新比赛比分 |
| `/stats/{event_id}` | GET | 获取统计数据 |

## 🔧 故障排查

### 问题 1：前端无法连接 API

- 检查 `API_BASE_URL` 是否正确
- 检查浏览器控制台是否有 CORS 错误
- 确保函数触发器配置了正确的请求方法

### 问题 2：数据库连接失败

- 检查环境变量是否正确配置
- 确认 TiDB Cloud 集群状态正常
- 检查函数计算日志（控制台 → 日志查询）

### 问题 3：多人编辑冲突

当前实现采用**最后写入优先**策略：
- 每 5 秒自动从云端拉取最新数据
- 如果两人同时编辑同一场比赛，后保存的会覆盖先保存的

建议：同一场比赛只由一人负责录入
